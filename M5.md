## 2. Complete Platform Architecture

The MedTrace AI platform follows a microservices-inspired monolithic architecture optimized for the hackathon but scalable for production.

### Layered Responsibilities

* **Frontend Layer (M3):** Streamlit container. Communicates strictly over REST and Server-Sent Events (SSE) to the API Layer.
* **API Layer (M5):** FastAPI application. Handles routing, schema validation, rate limiting, and authentication.
* **Integration Layer (M5):** Service abstractions shielding the API from direct database interactions.
* **Agent Layer (M2):** LangGraph state machine executed within FastAPI background tasks or async generators.
* **Graph Layer (M1):** Neo4j AuraDB. Connected via async Neo4j Python driver.
* **RAG Layer (M4):** ChromaDB container. Accessed via LangChain/Chroma wrappers.
* **Storage Layer (M5):** PostgreSQL container. Stores user sessions, historical graph snapshots (Time Machine), and audit logs.
* **Deployment Layer (M5):** Docker Engine and Docker Compose orchestrating the local network.

### Data Flow Diagram

```text
[Streamlit Frontend] 
       │ (REST / SSE)
       ▼
[FastAPI Gateway] ─── (Auth & Rate Limit) ─── [PostgreSQL: Users/Logs]
       │
       ├──► [Graph Service] ──────────► [Neo4j AuraDB: Digital Twin]
       │
       ├──► [Document Service] ───────► [ChromaDB: RAG Vector Store]
       │
       └──► [Agent Orchestrator] ─────► [LangGraph Engine] ──► [OpenAI/Claude API]
                  │
                  └──► [Event Bus] ───► [PostgreSQL: Time Machine Snapshots]

```

## 3. FastAPI Architecture

The FastAPI backend utilizes Domain-Driven Design (DDD) to keep the codebase modular.

### Internal Structure

* **API Gateway (`routers/`):** Defines HTTP endpoints, HTTP methods, and Pydantic response models.
* **Service Layer (`services/`):** Contains business logic (e.g., `AuditService`, `InvestigationService`).
* **Integration Layer (`clients/`):** HTTP and TCP clients for Neo4j, ChromaDB, and LLM providers.
* **Middleware Layer (`middleware/`):** Request correlation IDs, CORS, logging, and error catching.
* **Error Handling Layer (`exceptions.py`):** Custom exception classes mapped to specific HTTP status codes.

### Request Lifecycle

1. HTTP Request hits Uvicorn ASGI server.
2. Middleware attaches a `Trace-ID`.
3. Authentication dependency validates the JWT.
4. Router maps payload to Pydantic schema.
5. Service executes business logic (e.g., triggers an agent).
6. Result mapped back to Pydantic output schema.
7. Background task logs the action to PostgreSQL.
8. HTTP Response returned.

## 5. M5 ↔ M1 Integration Architecture (Neo4j)

FastAPI connects to Neo4j using the `neo4j` async Python driver. M5 abstracts this behind a Repository pattern.

### Implementation Details

* **Connection Pooling:** Initialized on `@app.on_event("startup")` and closed on `"shutdown"`.
* **Repository Layer:** `Neo4jRepository` class contains strictly parameterized Cypher queries provided by M1.
* **Safety:** Prevents Cypher injection attacks by strictly using `$` parameters, never f-strings.

```python
# Integration Pattern
async def get_device_subgraph(device_id: str, driver: AsyncDriver):
    query = "MATCH (d:Device {id: $id})-[r*1..2]-(n) RETURN d, r, n"
    async with driver.session() as session:
        result = await session.run(query, id=device_id)
        return await parse_graph_result(result)

```

## 6. M5 ↔ M2 Integration Architecture (LangGraph)

M5 bridges the synchronous API world with the asynchronous, stateful world of LangGraph agents.

### Streaming Architecture

FastAPI uses `StreamingResponse` to push LangGraph events to the frontend.

* M2 builds a compiled graph that yields state updates.
* M5 wraps this execution in an asynchronous generator.
* Every yielded state is converted to a JSON string prefixed with `data: ` (Standard SSE format).

```python
# Event Stream Pattern
async def agent_event_generator(complaint: str):
    async for event in m2_pipeline.astream({"raw_complaint": complaint}):
        yield f"data: {json.dumps(event)}\n\n"

@router.post("/agents/investigate")
async def trigger_investigation(payload: ComplaintPayload):
    return StreamingResponse(agent_event_generator(payload.text), media_type="text/event-stream")

```

## 10. WebSocket & SSE Architecture

While WebSockets offer bi-directional communication, **Server-Sent Events (SSE)** are strictly superior for the agent streaming use case because they operate over standard HTTP, simplifying Docker networking and preventing proxy timeout issues.

### SSE Message Schema

```json
{
  "event_id": "req_12345",
  "timestamp": "2026-06-23T21:46:13Z",
  "active_agent": "EvidenceAgent",
  "status": "Retrieving test protocol TR-018",
  "payload": null,
  "is_complete": false
}

```

* **Connection Management:** Handled natively by Starlette/FastAPI `StreamingResponse`.
* **Reconnection:** The UI handles automatic retries. M5 stateless architecture ensures no data locks if a connection drops.

## 11. Authentication & Security Architecture

For the hackathon demo, Auth is implemented via strict Middleware but mocked with hardcoded tokens to prevent live login failures.

### Security Layers

* **Tokens:** JWT (JSON Web Tokens) passed in the `Authorization: Bearer <token>` header.
* **RBAC:** FastAPI `Depends(RoleChecker(["admin", "auditor"]))`.
* **Secrets:** Managed exclusively via `.env` files. No hardcoded keys in the repository.

### Roles

* `RegulatoryManager`: Full read/write, triggers all agents.
* `Auditor`: Read-only graph access, triggers AuditShadow.
* `Engineer`: Read-write on Documents and Graph edges.

## 13. Regulatory Time Machine Infrastructure

M5 builds the backend capability to answer: "What did the graph look like 6 months ago?"

### Implementation

1. **Snapshot Trigger:** Nightly cron job or manual API call.
2. **Extraction:** M5 queries M1 to export the entire Neo4j node/edge list as a JSON object.
3. **Storage:** M5 saves this JSON payload into a PostgreSQL `JSONB` column, tagged with a timestamp and version hash.
4. **Replay:** When M3 requests historical data, M5 serves the PostgreSQL JSONB object instead of the live Neo4j data, instantly recreating the past state.

## 18. Failure & Recovery Architecture

M5 must ensure the demo never crashes.

### Strategies

* **Database Unavailable:** FastAPI retries connection 5 times with exponential backoff on startup.
* **LLM API Timeout:** If OpenAI/Claude times out, M5 intercepts the LangGraph error, stops the SSE stream gracefully, and yields an AI apology message: `{"error": "Investigation paused due to cognitive overload. Retrying..."}`.
* **Fallback Mode:** A `.env` toggle `DEMO_MODE=offline` forces the backend to bypass LLM calls and return perfectly formatted, pre-recorded responses for the exact demo script inputs.

---

## 19. Platform Performance Architecture

* **Caching:** `@alru_cache` applied to static graph queries (like fetching standard ISO frameworks) to eliminate redundant database calls.
* **Connection Pooling:** `asyncpg` and `neo4j` async drivers maintain connection pools to handle parallel agent queries instantly.
* **CORS:** Strictly configured to allow Streamlit connections and prevent pre-flight request lag.
